# AGENTS.md Support for Codex — Atomic Spec

> **Parent Brief:** `docs/briefs/2026-03-23-joysmith-cli-plugin.md`
> **Status:** Ready
> **Date:** 2026-03-23
> **Estimated scope:** 1 session / 2 files / ~100 lines

---

## What

When `joysmith init` scaffolds a project, also generate an `AGENTS.md` file alongside `CLAUDE.md` for Codex compatibility. AGENTS.md should contain the same behavioral boundaries and project context in the format Codex expects.

## Why

Joysmith should work with both Claude Code and Codex. Codex reads AGENTS.md (not CLAUDE.md) for project instructions. Generating both ensures users get the harness benefits regardless of which tool they use.

## Acceptance Criteria

- [ ] `npx joysmith init` creates AGENTS.md alongside CLAUDE.md
- [ ] AGENTS.md contains: behavioral boundaries, architecture overview, development commands, common gotchas
- [ ] AGENTS.md format follows Codex conventions (simpler structure than CLAUDE.md — see AGENTS_MD_TEMPLATE.md)
- [ ] If AGENTS.md already exists: improve it (same merge strategy as CLAUDE.md)
- [ ] Content is consistent between CLAUDE.md and AGENTS.md (same boundaries, same commands)
- [ ] AGENTS.md references CLAUDE.md as the canonical source for detailed instructions
- [ ] Build passes
- [ ] Tests pass

## Constraints

- MUST generate AGENTS.md from the same stack detection data as CLAUDE.md — no divergence
- MUST NOT duplicate everything from CLAUDE.md — AGENTS.md should be concise and reference CLAUDE.md for details
- SHOULD follow the existing AGENTS_MD_TEMPLATE.md structure from Joysmith templates
- SHOULD include a note that AGENTS.md is auto-generated by Joysmith and CLAUDE.md is the source of truth

## Affected Files

| Action | File | What Changes |
|--------|------|-------------|
| Create | `src/agents-md.ts` | Generate AGENTS.md content from stack info + boundaries |
| Modify | `src/init.ts` | Call agents-md generation as part of init flow |
| Create | `tests/agents-md.test.ts` | Test AGENTS.md generation |

## Approach

After generating CLAUDE.md content during init, extract the key sections (boundaries, commands, architecture summary) and format them into AGENTS.md using the AGENTS_MD_TEMPLATE structure. AGENTS.md is intentionally shorter — it contains the essentials and points to CLAUDE.md for the rest.

**Rejected alternative:** Generating AGENTS.md as an exact copy of CLAUDE.md. Codex has different conventions and a shorter file works better. Also, maintaining two identical files invites drift.

## Edge Cases

| Scenario | Expected Behavior |
|----------|------------------|
| Project only uses Codex, never Claude Code | AGENTS.md still generated, CLAUDE.md still generated (both are useful) |
| User has custom AGENTS.md with team-specific rules | Merge strategy — add Joysmith sections, preserve custom content |
| User deletes AGENTS.md after init | `npx joysmith upgrade` offers to re-create it |
